## Documentation Requirements

### Code Documentation
- All public interfaces must be documented with clear comments
- Use Swift DocC format for documentation comments
- Include parameter descriptions, return values, and throws clauses
- Document complex algorithms with explanations of methodology
- Add links to scientific research where applicable
- Use diagrams for complex flows (preferably as code with tools like Mermaid)

### README Requirements
- Clear project description with app purpose and key features
- Setup instructions with prerequisites and configuration steps
- Architecture overview with key components diagram
- Development workflow guidelines
- Testing approach
- Links to external documentation
- Screenshots of key app screens
- Troubleshooting section for common issues

### Developer Documentation
- Architecture decision records (ADRs) for significant choices
- Data flow diagrams for key processes
- Authentication and privacy implementation details
- HealthKit integration guide
- Detailed explanation of life impact calculation algorithms
- Detailed explanation of life projection algorithms
- UI component catalog with usage examples
- Extension points for future development
- Performance optimization guidelines

### User Documentation
- Clear explanations of health metrics and their impact
- Scientific basis for calculations
- Privacy policy in plain language
- Subscription benefits and management guide
- FAQ for common user questions
